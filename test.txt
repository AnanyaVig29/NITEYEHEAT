mplementation Plan

Start with a foundation phase, then build features in the order that gives the most value and the least risk.

Phase 1: Core tracking foundation

Add responsive coordinate normalization so gaze points stay accurate across screen sizes.
Add fixation filtering so noisy eye movement is reduced before any analytics or rendering.
Add dynamic calibration with a quick one-dot recalibration flow to correct drift without restarting the session.
Add privacy mode so video stays client-side and only gaze data or derived metrics are used.
Phase 2: Real-time visualization

Add saccade pathing using Canvas API so the app can draw the user’s gaze journey in order.
Add attention score so the UI can show a live focus meter.
Add blindness alerts so the system highlights important areas that never received gaze.
Add responsive scaling so the same session can be reviewed properly on desktop, tablet, and mobile.
Phase 3: Accessibility features

Add dwell-clicking using JavaScript timers for hands-free interaction.
Add gaze scrolling using the Window Scroll API for navigation by eye movement.
Add blink triggers using OpenCV.js or MediaPipe for gesture-based actions like Back or Refresh.
Phase 4: Analytics features

Add AOI statistics using geometry logic and SQL so you can measure time to first fixation on buttons and key areas.
Add session export using JSON and Blob Web APIs so researchers can download replayable session files.
Add better session summaries and per-page heatmap metrics for reporting.

Phase 5: Technical hardening

Cache live analytics results briefly to reduce latency.
Add event logging for calibration quality, fixation confidence, and dwell actions.
Add test coverage for coordinate mapping, fixation filtering, and session export.
